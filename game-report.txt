1. CORE GAME LOOP

CONCEPT: Real-time MOBA where AI bots and humans battle on the same battlefield.
Alliance (blue) vs Iron Horde (red). Continuous matches, auto-restart.

BROADCAST RATE: 8 state broadcasts per second to all WebSocket clients
MAP SIZE: 4800 x 2400 (world coordinates)
LANES: 3 (top Y=400-600, mid Y=1040-1360, bot Y=1800-2000)
VISION RADIUS: 400 units (fog of war)
DAY/NIGHT CYCLE: 120 seconds full cycle (60s day, 60s night)
WAVE SPAWN INTERVAL: Every 35 seconds
POSTGAME DELAY: 30 seconds between matches
MAX PLAYERS PER FACTION: 5 (10 total players per match)

5. COMBAT DAMAGE FORMULA
────────────────────────────────────────────────────────────────────────────────

  reduction = target.armor / (target.armor + 50)
  cheerMult = 1.25 if attacker faction has active rally buff, else 1.0
  finalDmg = max(1, floor(rawDmg * cheerMult * (1 - reduction)))

Example: 100 raw damage vs 15 armor target with no buff:
  reduction = 15 / (15+50) = 0.231
  finalDmg = 100 * 1.0 * (1 - 0.231) = 77

Notes:
- Minimum 1 damage always (cannot be reduced below)
- Last 5 attackers tracked per hero for assist credit
- Wave-scaled units: HP/damage * (1 + waveCount * 0.03) * eraMultiplier

8. WAVE PROGRESSION & ERA SYSTEM
────────────────────────────────────────────────────────────────────────────────

Waves spawn every 35 seconds from each barracks (alive only).
Each wave: 8 units per faction, distributed across all 3 lanes (top/mid/bot).

Per-wave scaling: stats * (1 + waveCount * 0.03)  [+3% per wave]

ERA PROGRESSION (additional multiplier on top of wave scaling):
  Wave 1-4   = Bronze Era   (1.0x multiplier)
  Wave 5-9   = Silver Era   (1.15x multiplier)
  Wave 10-14 = Gold Era     (1.30x multiplier)
  Wave 15-19 = Platinum Era (1.45x multiplier)
  Wave 20+   = Diamond Era  (1.60x multiplier)

WAVE VOTING (F1/F2/F3 keys):
  F1 — Vote for melee swarm (more footmen/warriors)
  F2 — Vote for ranged army (more archers/shredders)
  F3 — Vote for heavy siege (gryphons + ballistas / colossi)
  Vote resets after each wave spawns.

9. LANE / AI BEHAVIOR
────────────────────────────────────────────────────────────────────────────────

Units (auto-controlled):
- Spawn from barracks, march toward enemy base along their assigned lane
- Stop and attack any enemy in range
- Move at unit.speed * dt per tick
- Y-clamped to lane corridor (top/mid/bot)

Heroes (auto mode):
- Move toward enemy base
- Attack nearest enemy in range every attackCd ticks
- Use abilities when available and target in range
- Buy items when affordable (gold thresholds: 200, 250, 300, 600)
- Retreat behavior: heal at base if HP < 30%

Heroes (manual mode — DEFAULT for browser players):
- Player controls movement (WASD or right-click)
- AUTO-attack still active: hero attacks nearest enemy in range automatically
- Player triggers abilities (QWERT) and ultimate (Spacebar)
- Player can switch lanes (1/2/3) and fire base turret (V)

13. ELO RATING SYSTEM
────────────────────────────────────────────────────────────────────────────────

Standard ELO formula with K=32:
  expectedWin = 1 / (1 + 10^((loserElo - winnerElo) / 400))
  newWinner = round(winnerElo + 32 * (1 - expectedWin))
  newLoser  = round(loserElo + 32 * (0 - expectedLose))

ELO updates on hero-vs-hero kills only (not unit kills).
All agents start at ELO 1200.

RANK TIERS (UI display only):
  Bronze:   < 1200
  Silver:   1200-1399
  Gold:     1400-1599
  Diamond:  1600-1999
  Master:   2000+

15. SPECTATOR BETTING SYSTEM
────────────────────────────────────────────────────────────────────────────────

Pari-mutuel pool: spectators bet on Alliance OR Horde to win the match.

Odds calculation:
  total = alliance + horde
  alliance odds = (total / alliance).toFixed(1)
  horde odds = (total / horde).toFixed(1)

Payout: When match ends, all bettors on the winning faction split the
losing pool proportional to their bet size.

5% burn (planned, not yet on-chain): 5% of every losing pool is meant to
be burned permanently to the $WAR contract.

State resets between matches (no persistence yet).

CHEER / RALLY:
  Spectators can burn $WAR (currently virtual) to grant their faction
  +25% damage for 30 seconds.
  Total burned amount tracked per match.

PROP MARKETS (predictions):
- First Blood (which faction gets first kill)
- First Tower Falls (which faction destroys first tower)
- MVP Class (which class on the winning side has most kills)

17. RENDERING ARCHITECTURE
────────────────────────────────────────────────────────────────────────────────

Hybrid renderer:
- Phaser 3: terrain (Alliance grass / Horde Mordor), structures, castle
  silhouettes, lava cracks, dead trees, fire pits, towers
- HTML Canvas overlay (entity-canvas): heroes, units, names, HP/MP bars,
  damage numbers, ability FX, ultimate flash, kill rewards

Why hybrid: Phaser sprite rendering had persistent issues that took multiple
debug sessions. The pure canvas overlay for entities is simpler and more
reliable. They work together — entity canvas has pointer-events:none so
clicks pass through to Phaser for camera/click-to-move.

18. KNOWN BUGS & INCOMPLETE
────────────────────────────────────────────────────────────────────────────────

KNOWN BUGS:
- Bottom hero panel cards occasionally render with missing data on some
  heroes (defensive try/catch added but root cause unclear)
- Horizontal scrollbar still appears on some screen sizes despite multiple
  CSS overflow:hidden fixes
- Camera occasionally jitters when manual-mode hero changes direction
  rapidly
- Fog of war is server-computed but client doesn't render the dark overlay
  (only dims enemy sprites at 15% alpha when out of vision)
- "$WAR" UI references and contract addresses are placeholders (0x000...000)

INCOMPLETE / NOT YET IMPLEMENTED:
- $WAR ERC-20 token not deployed (UI shows "Coming Soon")
- King of the Hill on-chain drip — UI shows the calculated rate but no
  payouts happen
- Burn-on-loss prediction markets — 5% burn calculated but not on-chain
- Wallet connection is a stub (button exists, no functionality)
- Tournament/bracket system not built
- 1v1/3v3 ranked queue not built
- Ability tier upgrades work server-side but no UI to choose which ability
  to upgrade — they auto-upgrade randomly on level up
- Fog of war rendering on client is only entity-dimming, not a black mask
- Replay viewer page exists at /replay/:id but playback UI is basic
- Resurrection ability (Priest R) — code path exists but revive logic is
  incomplete (currently same as heal)

UNBALANCED / TUNING ISSUES:
- Knight tier 3 (Lv10): +40% +30% +30% on top of base 900 HP = ~1638 HP
  with 25 damage scaled. Knight becomes very tanky vs ranged DPS.
- Mage starts at only 450 HP — very fragile, dies fast in mid lane
- Priest base damage 15 is too low — Priest is essentially helpless solo
  if no allies nearby
- Siegemaster speed 60 is very slow — hard to escape ganks
- Gold income too low: passive 3g/2s = 90g/min. Items cost 200-600. Heroes
  rely heavily on kills (300g) and assists (75g) for items.
- Match length: with current scaling, matches average 8-15 minutes if no
  one team dominates. Postgame delay 30s feels long.
- Wave scaling: by wave 20 (Diamond Era), units have ~1.6x * 1.6x = 2.56x
  base stats. A Footman becomes 768 HP / 46 dmg. Fine for late game but
  early game waves trivially die to towers.
- Ultimate cooldowns (60-90s) feel about right but Priest's Divine
  Resurrection (90s) is extremely powerful — instant full team revive
  could be game-deciding.
- Holy Judgment (Knight ULT) deals 500 damage to ALL enemies — at higher
  levels with armor reduction this is still 200-400 per target. Possibly
  OP in team fights.
